## 1. 调研目标

在 OTTO 123 技术架构下，调研**少儿图形化编程**（类 Scratch）控制 Otto Robot 的完整方案，重点解决：

1. **编程工具选型**：直接用 Otto Blockly 还是自研 / 选其他方案
2. **积木设计**：面向初中生的动作编程积木如何定义
3. **软硬件通信**：Blockly 生成的动作指令如何传到机器人执行
4. **实时性**：课堂场景（15-20台设备）的真机预览延迟控制

### 2.1 Otto Blockly（官方方案）

**官网**：https://ottodiycn.mystrikingly.com/ （中文站）

**技术栈**：
- 基于 Google Blockly 10.x
- 支持 Arduino C++ 代码生成
- 支持蓝牙 / USB 两种方式下载程序

**优点**：
- 官方直接支持，和硬件完全匹配
- 已有丰富的 Otto 动作积木（走路、跳舞、唱歌等）
- 社区活跃，文档齐全

**缺点**：
- 界面是英文的（需汉化）
- 预设积木面向 8-12 岁儿童，初中生可能觉得太简单
- 蓝牙通信在课堂多设备场景有干扰问题
- Wi-Fi 版本的 Otto Blockly 是付费功能？

**与 OTTO 123 的匹配度**：
- ✅ 直接可用，架构文档 R14 已选用 Blockly
- ⚠️ 需要：汉化 + 适配 ESP32-S3 固件 + 支持 WebSocket 通信（架构选的是 MQTT/WebSocket，不是蓝牙）

**结论**：核心方案，但需要大幅定制

### 2.2 Mind+（国产优质方案）

**官网**：https://mindplus.cc/

**特点**：
- DFRobot 出品，国产精品
- 支持 Scratch 模式 / Arduino 模式 / Python 模式切换
- 支持 Arduino、micro:bit、ESP32、掌控板等
- 扩展库丰富（传感器、显示、通讯等）

**优点**：
- 中文界面，对国内用户友好
- 支持 Wi-Fi 通信（支持 MQTT、HTTP）
- 有教材和课程体系
- 实时模式（图形化积木直接控制硬件）和上传模式（生成代码）两种

**缺点**：
- 面向小学，界面风格偏幼
- 扩展积木开发文档不完善
- 闭源，二次开发受限

**与 OTTO 123 的匹配度**：
- ⭐⭐⭐⭐ 国产成熟，但需评估二次开发可行性

## 3. 技术实现路径分析

### 3.1 通信架构选择

架构文档已确定：
- **控制通道**：MQTT（低频，动作指令、状态上报）
- **媒体通道**：WebSocket（高频，语音流）

对于图形化编程，有两种模式：

**方案 A：实时预览模式**
```
Blockly 拖拽 → WebSocket 即时发送 → 机器人立即执行
```
- 优点：所见即所得，教学体验好
- 缺点：课堂多设备时 Wi-Fi 拥塞；舵机频繁响应有机械磨损
- 适用：调试阶段，1-2 台同时预览

**方案 B：批量执行模式**（架构 R15 已选）
```
Blockly 拖拽 → 生成动作序列 JSON → 点击"执行" → MQTT 发送 → 机器人执行
```
- 优点：课堂限流可控（最多 5 台同时执行），Wi-Fi 压力小
- 缺点：无法实时拖动预览
- 适用：正式编程课

**推荐**：采用方案 B，与架构 R15 一致。调试时提供"单步预览"开关。

---

### 3.2 积木设计建议

面向初中生（12-15 岁），需要比小学 Scratch 更丰富：

| 积木类别 | 示例积木 | 说明 |
| --- | --- | --- |
| **基础动作** | 走路（步数/方向/速度） | Otto 标准动作 |
| | 跳舞（选择预设舞步） | 内置 10+ 预设 |
| | 挥手 / 点头 / 摇头 | 基础肢体 |
| **舵机控制** | 设定 [关节] 角度 [0-180] | 精细控制 |
| | 动作序列录制 | 手动引导后保存 |
| **控制流** | 循环 N 次 | 重复执行 |
| | 如果...那么... | 条件分支 |
| | 等待 N 秒 | 时序控制 |
| **传感器** | 当 [超声波] 距离 < [10]cm | 触发条件 |
| | 当 [陀螺仪] 检测到 [摔倒] | 事件触发 |
| **音乐** | 播放音乐 [选择] | 内置 + 自定义 |
| | 设定 BPM [N] | 节拍控制 |
| **高级** | AI 生成动作 [文字描述] | R18 云端能力 |
| | 变量 / 函数 | 编程进阶 |

---

### 3.3 代码生成目标格式

Blockly 积木生成的中间格式建议用 JSON：

```json
{
  "version": "1.0",
  "actions": [
    { "type": "walk", "params": { "steps": 2, "direction": "forward", "speed": 50 } },
    { "type": "delay", "params": { "ms": 500 } },
    { "type": "servo", "params": { "part": "left_arm", "angle": 90 } },
    { "type": "music", "params": { "track": "happy", "bpm": 120 } }
  ]
}
```

机器人固件解析 JSON，转换为舵机控制序列执行。

## 4. 推荐方案

### 4.1 核心选择：基于 Otto Blockly 深度定制

**理由**：
1. 架构 R14 已确定使用 Google Blockly，Otto Blockly 是官方配套
2. 完全兼容 ESP32-S3 硬件
3. 可在开源基础上进行：
   - 汉化界面
   - 增加 ESP32 Wi-Fi 通信积木（替代蓝牙积木）
   - 增加初中生适用的进阶积木
   - 对接 OTTO 123 的 MQTT/WebSocket 通信协议

### 4.2 备选：Mind+（如果 Otto Blockly 二次开发遇到瓶颈）

- 国产方案，本地化和技术支持好
- 原生支持 Wi-Fi MQTT
- 有现成的课程体系可借鉴
