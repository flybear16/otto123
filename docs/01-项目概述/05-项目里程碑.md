# 项目里程碑

> 基于 PRD v3-final 三期演进规划（机构内部赛 → 机构间联赛 → 线上社区）和愿景文档短/中/长期目标，制定项目里程碑路线图。

---

## 总体路线图

```
Phase 1 (MVP)              Phase 2                   Phase 3
核心功能 + 试点验证    →    机构间联赛 + 规模化   →    线上社区 + 生态建设

|─── 0 ~ 6 个月 ───|─── 7 ~ 18 个月 ───|─── 19 ~ 36 个月 ───|
```

---

## Phase 1：MVP — 核心功能开发与试点验证

### 时间范围

**第 0 ~ 6 个月**

### 目标

完成 OTTO 机器人教育平台核心功能开发，在 2-3 家试点机构完成验证，确认产品价值和商业模式可行性。

### 关键交付物

| 交付物 | 说明 | 优先级 |
|--------|------|--------|
| OTTO 机器人硬件套件 v1 | ESP32-S3 + 8MB PSRAM + ES8311，基础版 ≤ 250 元/台 | P0 |
| 固件核心功能 | 语音唤醒（ESP-SR）、MQTT 通信、舵机控制、OTA 升级 | P0 |
| AI 语音交互 | 接入大模型，自然语言对话 + 语音控制动作 | P0 |
| 网页端配置工具 | 可视化参数调整、舵机校准、设备管理 | P0 |
| 图形化编程工具 | Blockly 拖拽编程，28+ 预设动作积木 + 3D 模拟预览 | P0 |
| 内置课程体系 | 12 节互动教程（入门 4 节 + 进阶 4 节 + 高级 4 节） | P0 |
| 多角色后台系统 | 管理员 / 教师 / 家长三级权限，内容推送与安全审计 | P0 |
| 机构内部赛平台 | 外观选美 + 动作技能展示（手动组/AI 组）+ 功能创新赛 | P1 |
| DIY 外壳体系 | 开源 STL 模板 + 3 种预设外壳方案 | P1 |
| 教师教案文档 | 配套 12 节课的教师教案和使用指南 | P1 |

### 成功指标

| 指标 | 目标值 | 验证方式 |
|------|--------|----------|
| 试点机构数 | 2-3 家晚托班/学校 | 合作协议签署 |
| 覆盖学生数 | 100+ 名学生 | 平台注册数据 |
| 学习曲线 | 学生 2 节课后独立完成简单动作编程 | 课堂观察记录 |
| 作品产出 | 学生 8 节课后完成自定义外壳 + 编程作品 | 作品提交数据 |
| 系统可靠性 | 课堂中断 < 1 次/月/机构 | 故障日志统计 |
| 用户满意度 | 教师满意度 ≥ 80%，学生满意度 ≥ 85% | 问卷调查 |
| 出勤率影响 | 引入后出勤率有可见提升 | 机构运营数据 |

## Phase 2：规模化 — 机构间联赛与区域扩展

### 时间范围

**第 7 ~ 18 个月**

### 目标

在试点验证成功的基础上，扩大机构覆盖范围，上线机构间联赛功能，建立区域品牌影响力。

### 关键交付物

| 交付物 | 说明 | 优先级 |
|--------|------|--------|
| 机构间联赛平台 | 不同机构线上/线下竞赛，排行榜和积分体系 | P0 |
| 扩展模块支持 | 摄像头、大屏显示、额外舵机等即插即用模块 | P0 |
| AI 辅助动作生成 | 自然语言描述 → AI 生成舵机序列，可手动微调 | P1 |
| 音乐配合功能 | 动作与音乐节拍对齐，本地播放 | P1 |
| 数据分析看板 | 机构运营数据、学生参与度趋势、出勤率分析 | P0 |
| 多人协作优化 | 用户快速切换、课堂模式增强、使用时长智能提醒 | P1 |
| 扩展课程体系 | 竞赛集训课程、进阶 DIY 课程 | P1 |
| 硬件 v2 迭代 | 根据试点反馈优化硬件设计，降低故障率 | P2 |

### 成功指标

| 指标 | 目标值 | 验证方式 |
|------|--------|----------|
| 覆盖机构数 | 10+ 家机构 | 合作协议签署 |
| 覆盖学生数 | 1000+ 名学生 | 平台注册数据 |
| 联赛参与 | 至少 10 个机构同时参与联赛 | 竞赛平台数据 |
| 竞赛获奖 | 学生在市级/省级科技竞赛中获奖 | 获奖证书 |
| 续课率 | 引入后机构续课率提升 ≥ 15% | 机构运营数据 |
| 系统并发 | 10 个机构（150-200 台设备）同时在线，P95 延迟达标 | 性能监控 |
| 设备在线率 | 月度设备在线率 ≥ 95% | 设备管理后台 |

---

## Phase 3：生态建设 — 线上社区与课程输出

### 时间范围

**第 19 ~ 36 个月**

### 目标

构建线上创客社区，实现课程授权输出，从"产品"升级为"平台 + 生态"，成为区域领先的创客教育品牌。

### 关键交付物

| 交付物 | 说明 | 优先级 |
|--------|------|--------|
| 线上创客社区 | 学生上传/下载/点赞/评论作品，社区运营 | P0 |
| 课程授权体系 | 向外部机构输出课程授权，合作 100+ 学校/机构 | P0 |
| 师资培训认证 | 建立师资培训和认证体系，标准化教师输出 | P0 |
| 开发者生态 | 第三方扩展模块支持，社区贡献机制 | P2 |
| 多城市扩展 | 从杭州向周边城市扩展，输出课程和运营模式 | P1 |
| 品牌升级 | 从"性价比"到"专业"的品牌形象升级 | P1 |
| 竞赛品牌化 | 打造 OTTO 专属竞赛 IP，形成年度赛事 | P1 |

### 成功指标

| 指标 | 目标值 | 验证方式 |
|------|--------|----------|
| 覆盖机构数 | 100+ 学校/机构 | 合作数据 |
| 覆盖学生数 | 10000+ 名学生 | 平台注册数据 |
| 社区活跃度 | 月活用户 ≥ 5000，日均作品上传 ≥ 50 | 社区数据 |
| 课程授权收入 | 课程授权成为主要收入来源之一 | 财务数据 |
| 品牌认知度 | 区域内创客教育品牌认知度 Top 5 | 市场调研 |
| 竞赛规模 | OTTO 专属赛事覆盖 ≥ 50 所学校 | 赛事数据 |

## 风险与依赖

| 风险项 | 影响阶段 | 应对措施 |
|--------|----------|----------|
| Otto DIY 开源项目维护停滞 | 全阶段 | 关注社区动态，必要时 Fork 独立维护 |
| 大模型 API 成本上升或不可用 | Phase 1-2 | 支持多模型切换（Qwen/DeepSeek/GLM），不绑定单一供应商 |
| 试点机构验证失败 | Phase 1 | 设置 2-3 家试点分散风险，快速迭代产品 |
| 硬件供应链不稳定 | Phase 1-2 | 核心元器件（ESP32-S3、PSRAM）选择成熟方案，备选供应商 |
| 竞品进入低价位市场 | Phase 2-3 | 加速占领市场，通过开源生态和竞赛体系建立壁垒 |
| 政策变化影响素质教育 | 全阶段 | 产品符合 STEAM 教育政策方向，保持灵活性 |
