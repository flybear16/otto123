# 用户画像与非用户分析

> 一堂五步法 · 第一步：发现真实需求
> 分析水平：L2（主动拆解）——不仅描述"谁用"，更拆解"谁不用"、角色间的关系链与优先级、需求与 PRD 的映射关系。
> 产品定位：**高端可定制**（套件 800-1200 元，课时 300-500 元）

---

## 一、学生画像（12-15 岁初中生）

### 1.1 角色描述

在晚托班或学校社团中使用 OTTO 机器人的初中生。他们处于青春期早期，好奇心强，对新鲜科技事物感兴趣，但也容易因难度过高而放弃。课堂上通常 2-3 人共用一台机器人，协作与竞争并存。

### 1.2 核心诉求

| 诉求维度 | 具体期望 |
|----------|----------|
| 趣味性 | 机器人要好玩，能跳舞、能对话，不像上课 |
| 成就感 | 能做出属于自己的作品，得到同学和老师的认可 |
| 社交性 | 能和同学比拼、分享，竞赛中有展示机会 |
| 创造力 | 能自定义机器人外观和动作，表达个性 |
| 易上手 | 不需要太多基础知识就能开始玩 |

### 1.3 使用场景

- **课间互动**：晚托班课间用语音和机器人聊天、让它跳舞逗乐
- **兴趣课堂**：每周 1-2 节机器人课程，学习动作编程和 DIY 外壳
- **竞赛备赛**：为机构内部赛准备参赛作品（外观选美、动作技能、功能创新）
- **小组协作**：2-3 人一组共同完成一个编程作品，分工配合

### 1.4 痛点分析

- 现有编程课太枯燥，只有屏幕没有实物，缺乏"造物"的满足感
- 商业机器人套件价格高，自己买不起，只能在学校用
- 学习门槛高，很多创客教育产品需要先学大量理论知识才能上手
- 作品完成后缺乏展示和交流的平台，缺少持续学习的动力

### 1.5 期待收益

> "我能造一个独一无二的机器人，还能让它跳舞、陪我聊天、帮我赢得比赛。"

- 获得动手实践的机会，从"消费者"变成"创造者"
- 在竞赛中获得成就感和自信
- 自然学习编程思维和工程知识，为未来科技素养打下基础

### 1.6 量化属性

| 属性 | 数据 |
|------|------|
| 杭州目标人群 | ~15 万初中生（2025 年在校生规模） |
| 付费意愿 | 不直接付费，但 enthusiasm 影响家长决策 |
| 决策周期 | 无决策权，体验后即时反馈 |

## 二、教师画像（晚托班/社团授课教师）

### 2.1 角色描述

负责在晚托班或社团课中教授机器人课程的教师。可能是机构的全职教师，也可能是兼职的理工科大学生。技术背景参差不齐，部分教师对机器人和编程并不熟悉，需要低门槛的教学工具和现成的课程资源。

### 2.2 核心诉求

| 诉求维度 | 具体期望 |
|----------|----------|
| 易教学 | 有现成教案和互动教程，不需要自己备课 |
| 课堂可控 | 能管理学生使用行为，课堂不混乱 |
| 效果可见 | 能展示教学成果（学生作品、竞赛成绩） |
| 低门槛 | 不需要深厚的编程或硬件知识就能上手 |
| 持续支持 | 遇到问题能快速获得技术支持 |

### 2.3 使用场景

- **课前准备**：登录后台配置课程内容，推送学习资源到班级设备
- **课堂授课**：使用平台内置的 12 节互动教程授课，引导学生完成实操
- **课堂管理**：查看学生使用时长和互动记录，设置课堂模式和使用轮换
- **竞赛组织**：在平台内组织班级/机构内部赛，使用评分模板进行评审
- **家校沟通**：通过平台向家长推送学生课堂表现摘要

### 2.4 痛点分析

- 传统创客教育备课负担重，需要自己摸索教学内容和实验方案
- 课堂中学生水平差异大，难以兼顾每个学生的进度
- 缺乏标准化教学工具，设备管理和维护占用大量时间
- 学生使用电子设备时难以监控内容安全，存在管理风险

### 2.5 期待收益

> "我只需要打开平台、按照教程引导，就能上好一节精彩的机器人课。"

- 降低备课成本，有现成课程体系直接使用
- 课堂管理工具帮助掌控教学节奏
- 通过竞赛和作品展示提升教学成就感
- OTA 升级和设备诊断减少维护负担

### 2.6 量化属性

| 属性 | 数据 |
|------|------|
| 杭州目标人群 | ~3000-5000 名（晚托班+社团教师总量估算） |
| 付费意愿 | 不直接付费，但推荐/否定影响机构采购决策 |
| 决策周期 | 体验后 1-2 周形成判断 |

## 三、机构管理者画像（晚托班/培训机构运营方）

### 3.1 角色描述

晚托班或培训机构的负责人，关注招生、运营成本和差异化竞争。他们需要一套能帮助机构在同类竞品中脱颖而出的特色课程，同时控制投入成本。在高端定位下，管理者追求的不是最低价，而是"高感知价值 + 可控成本"。

### 3.2 核心诉求

| 诉求维度 | 具体期望 |
|----------|----------|
| 招生差异化 | 机器人课程成为招生亮点，区别于其他晚托班 |
| 成本可控 | 硬件和课程总投入在可接受范围内，ROI 可预期 |
| 运营高效 | 设备管理、教师管理、课程管理有系统化工具 |
| 数据驱动 | 能看到运营数据（出勤率、学生参与度、续课率） |
| 品牌提升 | 通过竞赛获奖和作品展示提升机构品牌形象 |

### 3.3 使用场景

- **采购决策**：评估 OTTO 套件成本和课程价值，决定引入数量
- **系统部署**：注册管理员账号，录入设备，配置网络环境
- **日常运营**：查看设备在线率、课程覆盖率、学生参与度趋势
- **师资管理**：管理教师账号权限，查看各班级教学数据
- **招生宣传**：用学生竞赛作品和机器人课堂实况作为招生素材

### 3.4 痛点分析

- 晚托班同质化严重，缺乏差异化竞争力，招生越来越困难
- 高端创客教育套件（如乐高）成本太高，中小机构难以承受
- 引入新课程后教师培训周期长，见效慢
- 缺乏数据化的运营工具，难以科学评估课程效果

### 3.5 期待收益

> "花很少的钱就能拥有一套特色课程，学生喜欢、家长认可、招生不愁。"

- 低成本打造机构差异化竞争力
- 系统化工具降低运营管理成本
- 竞赛体系为学生提供出口，提升续课率
- 数据看板支持科学决策

### 3.6 量化属性

| 属性 | 数据 |
|------|------|
| 杭州目标机构 | ~200 所初中 + ~1000 家课后服务机构 |
| 付费意愿 | 学校 4000 元/班/学期；高端机构 300-500 元/课时 |
| 决策周期 | 2-3 个月（评估 → 试用 → 签约 → 部署） |

## 四、家长画像

### 4.1 角色描述

关注孩子学习和成长的学生家长。他们希望孩子在晚托班不仅能完成作业，还能学到有价值的东西。同时，他们非常关注孩子使用电子设备时的内容安全和隐私保护。在高端定位下，家长愿意为"看得见的成长"支付溢价。

### 4.2 核心诉求

| 诉求维度 | 具体期望 |
|----------|----------|
| 学习有价值 | 孩子学的东西对未来有帮助，不是纯娱乐 |
| 内容安全 | 孩子和机器人对话的内容健康安全 |
| 了解动态 | 能了解孩子在课堂上的表现和学习进展 |
| 适度引导 | 能给孩子推送鼓励和提醒，参与孩子的学习 |
| 性价比高 | 课程价格合理，物有所值 |

### 4.3 使用场景

- **绑定账号**：通过机构分配的绑定码扫码绑定孩子账号
- **查看动态**：在手机端查看孩子使用时长和对话摘要
- **内容推送**：给孩子推送鼓励语、每日提醒等个性化内容
- **安全通知**：收到内容安全告警时及时了解情况并配合处理
- **作品欣赏**：查看孩子在竞赛中的作品和成绩

### 4.4 痛点分析

- 不清楚孩子在晚托班到底学到了什么，缺乏透明的反馈渠道
- 担心孩子使用带 AI 功能的设备时接触到不健康内容
- 想参与孩子的学习但缺乏便捷的工具和途径
- 很多编程/机器人课程价格虚高，难以判断是否值得

### 4.5 期待收益

> "我能随时了解孩子的学习情况，知道他在安全和有益的环境中成长。"

- 透明的学习反馈机制，了解孩子每节课的收获
- 多层内容安全过滤，让孩子在健康环境中使用
- 便捷的推送功能，参与孩子的学习过程
- 高性价比的教育产品，花更少的钱获得更好的体验

### 4.6 量化属性

| 属性 | 数据 |
|------|------|
| 杭州目标人群 | ~15 万初中生对应的家长群体 |
| 付费意愿 | 3000-5000 元/学期（优质 STEM 教育） |
| 决策周期 | 1-2 周（学校推荐后快速决策） |

## 六、非用户分析（谁不是我们的用户）

> 明确"谁不是用户"和"谁是用户"同样重要。以下群体被排除不是因为产品不够好，而是经济模型或产品形态不匹配。

### 6.1 12 岁以下儿童

| 维度 | 分析 |
|------|------|
| **排除原因** | 硬件操作复杂度超出独立使用能力；认知发展阶段不同，需要更简化的交互方式；安全要求更高（物理安全 + 内容安全） |
| **PRD 边界** | PRD 明确排除 6 岁以下（6.范围边界），但实际独立使用的合理边界是 12 岁 |
| **替代建议** | 如未来拓展低龄市场，需重新设计交互界面、课程体系和安全策略 |

### 6.2 纯 C2C 家庭消费者

| 维度 | 分析 |
|------|------|
| **排除原因** | 获客成本 >300 元/人，转化率 <10%，LTV 无法覆盖 CAC；家庭使用场景缺乏课堂中的社交/竞争动力学，用户粘性低 |
| **数据来源** | _archive/needs/opportunity-analysis-v1.md 3.1 排除清单 |
| **替代建议** | C2C 仅作为机构模式的延伸（学生体验后向家长推荐家庭版），不作为独立获客渠道 |

### 6.3 已采购乐高 SPIKE 的学校

| 维度 | 分析 |
|------|------|
| **排除原因** | 沉没成本高（已有设备投入 + 教师培训 + 课程体系）；乐高品牌忠诚度强，切换意愿低；同时维护两套系统增加教师负担 |
| **替代建议** | 聚焦尚未采购高端解决方案的学校，以及预算有限但追求差异化的中小机构 |

### 6.4 无课后服务预算的学校

| 维度 | 分析 |
|------|------|
| **排除原因** | 零预算意味着无论产品多优秀都无法成交；需要区分"无预算"和"预算未分配"——后者是潜在机会 |
| **替代建议** | 优先对接已有或计划分配课后服务预算的学校；关注政策性拨款窗口期 |

### 6.5 纯线上教育机构

| 维度 | 分析 |
|------|------|
| **排除原因** | OTTO 的核心价值是硬件动手体验，纯线上机构缺乏物理基础设施（教室、Wi-Fi、设备存放）；产品形态不匹配 |
| **替代建议** | 如果机构有线下校区或计划拓展线下，可作为潜在客户跟进 |

## 七、用户关系链与优先级

### 7.1 角色关系链

```
决策链：学校校长 → 教研组长 → 教师 → 学生
支付链：家长付费 → 学校/机构代收 → OTTO 收入
影响链：学生体验 → 家长满意度 → 教师推荐 → 机构续约
```

> 关键洞察：**学生是产品的使用者，但不是购买者；家长是最终付费者，但不是决策者；机构管理者是决策者，但不是使用者。** 产品必须在四个角色之间建立正向循环。

### 7.2 优先级排序

| 优先级 | 角色 | 定位 | 核心理由 |
|--------|------|------|----------|
| **P0** | 机构管理者 | 决策者 | 控制预算和采购决策。没有他们的认可，产品无法进入任何课堂 |
| **P1** | 教师 | 实施者 | 决定日常产品体验质量。教师用不好，产品在课堂上就会失败 |
| **P2** | 家长 | 付费者/评估者 | 最终付费方（直接或通过学费）。家长满意度驱动续费和口碑 |
| **P3** | 学生 | 终端用户 | 产品必须对 students 有吸引力，但他们没有购买力。他们的热情影响家长和教师 |

### 7.3 需求优先级矩阵

| 用户需求 | 主要角色 | 对应 PRD 需求 | 优先级 |
|----------|----------|---------------|--------|
| 有趣好玩的体验 | 学生 | R1-R5（语音交互）、R14-R19（动作编程） | Must-have |
| 易教学、低门槛 | 教师 | R6-R9（配置工具）、R17（课程体系）、R24-R30（管理与课堂管控） | Must-have |
| 招生差异化 | 机构管理者 | R10-R13（DIY 扩展）、R20-R23（竞赛平台） | Must-have |
| 内容安全 | 家长 | R26（安全过滤）、R27（使用监控） | Must-have |
| AI 语音交互 | 学生 | R1-R5 | Must-have（核心差异化） |
| 竞赛体系 | 机构管理者/学生 | R20-R23 | Phase 2 |
| 线上社区 | 学生 | R22 | Phase 3 |

## 八、定位校验

> 高端可定制定位（套件 800-1200 元，课时 300-500 元）下的用户适配性校验。

| 角色 | 与高端定位的匹配度 | 校验结论 |
|------|-------------------|----------|
| 机构管理者 | 高——高端定位正好解决"差异化"诉求，高课时费支撑高硬件投入 | 匹配，需强化"独特性"而非"低价"叙事 |
| 教师 | 中——教师不直接关心定价，但高定位意味着更高的教学标准和家长期望 | 需确保课程质量匹配价位 |
| 家长 | 高——愿意为优质教育付费，300-500 元/课时在 STEM 赛道属中高端可接受区间 | 匹配，需透明展示学习成果 |
| 学生 | 中低——学生不关心价格，但高定位意味着更复杂的功能，可能影响易上手性 | 需平衡"高端功能"与"低门槛体验" |
