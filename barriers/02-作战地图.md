## 二、客户分群与关键数据

| 属性 | 数据 |
|------|------|
| **目标人群** | 杭州 ~1200 所初中 + ~1000 家课后服务机构 |
| **付费意愿（WTP）** | 学校合作：¥4,000+/班/学期；机构植入：¥300-500/月/人 |

## 三、决策链与影响路径

```
决策链（学校模式）：

家长（付费者）
  │
  │ 满意 / 推荐
  ▼
学生（用户）──体验──▶ 教师（实施者）
                            │
                            │ 推荐 / 反馈
                            ▼
                       校长/主任（决策者）
                            │
                            │ 拍板
                            ▼
                       采购决策 → OTTO 进入

影响链（机构模式）：

学生体验好 ──▶ 家长续费 ──▶ 机构管理者决策续约/扩量
     │
     │ 课堂反馈
     ▼
   教师推荐
```

**关键结论**：四个角色形成正向循环——学生热情 → 课堂活力 → 教师认可 → 家长付费 → 机构决策 → 更多学生受益。

## 四、渠道分布与优先级

### 4.1 渠道分层

```
高优先级（主动出击）
├── 科技示范校 / 创新教育实验校
├── 私立/国际学校（决策快、预算弹性）
├── 已开设创客/机器人社团的学校（需求已被教育）
└── 课后服务已招标的机构（预算已确认）

中优先级（等待机会）
├── 普通公立初中（需等待政策/预算窗口）
├── 学科类培训机构转型（双减受益者）
└── 社区青少年活动中心

低优先级（暂不投入）
├── 纯C端家长（获客成本 >300 元，转化率 <10%）
└── 无课后服务预算的学校（无论产品多好都无法成交）
```

## 六、决策触发器与验证节点

### 6.1 B 端决策触发器

| 触发条件 | 预期反应 | 验证问题 |
|---------|---------|---------|
| 校长/主任主动关注科技教育 | 进入深度沟通 | 他们的科技教育愿景是什么？ |

## 八、MVP 验证路径建议

**推荐优先级**：晚托班（轻）→ 学校社团课（量）→ C端周末/竞赛（利润）

### Phase 1：晚托班验证（0-3 个月）

| 目标 | 验证假设 |
|------|---------|
| 3-5 所晚托班合作落地 | 轻资产模式可行性 |
| 100+ 学生持续使用 | 产品体验达标 |
| 家长续费率 >70% | C端付费意愿真实 |

**为什么先做晚托班**：投入小（5-10 套设备起）、验证快（1-2 个月见数据）、模式轻（按月结算无大账期）

### Phase 2：学校社团课（3-6 个月）

| 目标 | 验证假设 |
|------|---------|
| 2-3 所学校开课 | B端决策链可行 |
| 竞赛队伍组队参赛 | 竞赛体系可行性 |
| 口碑传播带来主动咨询 | 产品竞争力验证 |

### Phase 3：C 端规模化（6-18 个月）

| 目标 | 验证假设 |
|------|---------|
| 品牌知名度建立 | 可持续获客 |
| 竞赛成绩证明效果 | 升学/出口价值 |
| 家长主动推荐率 >50% | 口碑壁垒形成 |
