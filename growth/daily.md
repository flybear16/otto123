# 日常运营 - 每日SOP检查表

> 版本: v1.0 | 创建日期: 2026-04-04 | 作者: OTTO 运营团队

## 一、每日运营时间表

### 1.1 授课日流程

```
课前准备（提前30分钟）:
├── 场地检查
│   ├── 照明、空调正常开启
│   ├── 地面清洁，无障碍物（机器人行走安全）
│   └── 家长休息区整理
├── 设备准备
│   ├── 取出当日所需机器人套件，放置到操作台
│   ├── 每台通电测试：唤醒响应、动作执行、语音交互（R1-R5）
│   ├── 检查紧急停止按钮功能（R30）
│   ├── 教学平板开机，确认网络连接正常
│   └── 检查 Wi-Fi AP 状态，确认 5GHz 信号稳定
├── 教学准备
│   ├── 教案/课件确认
│   ├── 学生分组方案确认（2-3 人/组）
│   └── 签到表、学生档案准备
└── 课前会议（教师+运营，5分钟）
    ├── 确认今日课程内容
    ├── 通报请假/缺勤学生
    └── 特殊事项提醒

课中执行（90-120 分钟标准课时）:
├── 签到接待（10 分钟）
│   ├── 学生签到、核实人数
│   ├── 请假学生家长确认
│   └── 引导学生就位
├── 教学环节
│   ├── 课程导入（5-10 分钟）
│   ├── 知识讲解与演示（15-20 分钟）
│   ├── 学生动手操作（40-50 分钟）
│   │   ├── 按组轮流操作机器人
│   │   ├── 使用时长提醒：每人 15 分钟轮换（R29）
│   │   └── 教师巡场指导、拍照记录
│   └── 作品展示与总结（10-15 分钟）
├── 课堂管理
│   ├── 关注学生参与度（R27 参与度预警）
│   ├── 维护课堂纪律
│   └── 异常情况处理（设备故障、学生冲突等）
└── 家长同步
    ├── 课堂照片实时发家长群
    └── 课后立即发送课堂总结

课后收尾（15-20 分钟）:
├── 设备收纳
│   ├── 机器人关机，检查外观有无损坏
│   ├── 放回充电柜，接入充电
│   ├── 教学平板关机、归位
│   └── 故障设备标记并记录（R32）
├── 场地复原
│   ├── 操作台整理清洁
│   ├── 座椅归位
│   └── 垃圾清理
└── 教学记录
    ├── 填写教学日志（见下方模板）
    ├── 更新设备使用记录
    └── 学生表现记录
```

### 1.2 非授课日流程

```
非授课日（周一/周五）:
├── 上午
│   ├── 设备全面检查与维护
│   ├── 故障设备维修/送修
│   ├── 教案准备与优化
│   └── 教学研讨（团队内部）
├── 下午
│   ├── 下周课程准备
│   ├── 物料补充（打印资料、耗材等）
│   ├── 家长沟通/回访
│   └── 运营数据分析（出勤率、续费进度等）
└── 晚上
    ├── 家长群互动答疑
    └── 周报撰写与发送
```

## 三、每周运营节奏

```
周一（运营日）:
├── 上午: 设备全面检查 + 故障设备维修
├── 下午: 本周课程教案最终确认
├── 傍晚: 本周排课确认、通知家长
└── 晚上: 家长群发送本周课程预告

周二-周四（授课日）:
├── 按每日运营时间表执行（见第一节）
├── 当天: 课后发送课堂照片和作业到家长群
└── 记录: 每日教学日志归档

周五（运营日）:
├── 上午: 本周教学总结（填写周教学汇总表）
├── 下午: 下周课程准备 + 物料盘点
├── 傍晚: 设备充电确认、网络设备检查
└── 晚上: 家长群发送本周学习总结 + 优秀作品展示

周六-周日（活动日）:
├── 体验课（如有预约）
├── 竞赛辅导（如有安排）
├── 在线答疑（家长群/私信）
└── 个别学生补课/辅导（有需要时）
```

| 项目 | 填写内容 |
|------|----------|
| 本周授课 | 日期、班级、主题 |
| 出勤统计 | 本周总出勤率、缺勤原因汇总 |

| 教学反馈 | 学生掌握情况、教学难点、改进建议 |
| 家长反馈 | 投诉/建议/好评汇总 |
| 下周计划 | 课程安排、待办事项 |

## 四、月度运营检查清单

- [ ] 上月运营数据汇总（出勤率、满意度、续费率）

## 五、设备维护检查表

### 5.1 每日检查（授课前）

| 检查项 | 标准 | 操作 |
|--------|------|------|
| 外观 | 无明显损坏、螺丝无松动 | 目视检查 |
| 开机 | 正常启动，指示灯正常 | 通电测试 |
| 语音唤醒 | 唤醒响应 < 500ms（PRD 4.1） | 呼叫测试 |
| 动作执行 | 预设动作正常，无异常噪音 | 执行测试 |
| 语音对话 | AI 对话首字响应 < 2 秒（PRD 4.1） | 对话测试 |
| 编程连接 | 平板可正常连接和发送指令 | 编程测试 |
| 紧急停止 | 按下后舵机立即释放力矩（R30） | 按钮测试 |
| 电量 | >= 80%（课前充满电） | 电量查看 |

### 5.2 每周维护

| 维护项 | 频率 | 操作说明 |
|--------|------|----------|
| 舵机检查 | 每周 | 检查舵机是否有卡顿、异响，必要时校准（R7） |
| 充电口清洁 | 每周 | 检查充电触点是否有氧化/灰尘 |
| 固件更新 | 按需 | 检查是否有新版本，通过 OTA 分批升级（R8） |
| Wi-Fi 信号测试 | 每周 | 测试各操作台位置信号强度 |
| 外壳检查 | 每周 | 检查 3D 打印外壳是否有裂纹/变形（R10） |
| 扩展模块检查 | 按需 | 检查扩展接口是否松动（R12-R13） |

### 5.3 每月维护

| 维护项 | 频率 | 操作说明 |
|--------|------|----------|
| 深度清洁 | 每月 | 外壳擦拭、关节处除尘 |
| 电池健康检查 | 每月 | 充放电循环测试，电池衰减 >20% 更换 |
| SD 卡/存储检查 | 每月 | 音乐文件完整性检查（R19） |
| 备用设备轮换 | 每月 | 备用机与在用机轮换，确保备用机可用 |
| 工具盘点 | 每月 | 维修工具、螺丝、配件库存盘点 |
| 故障统计 | 每月 | 汇总本月故障类型和频次，分析趋势 |

### 5.4 设备故障处理流程

```
设备故障处理:
├── 课堂中故障
│   ├── 轻微故障（单台设备）
│   │   ├── 现场快速修复（重启、重新连接）
│   │   ├── 无法修复 → 取备用机替换（R33）
│   │   └── 课后登记故障，安排维修
│   └── 严重故障（多台设备/网络中断）
│       ├── 切换至离线模式教学（R5）
│       ├── 通知管理员/技术支持
│       └── 记录故障详情（错误码、现象、时间）
├── 故障上报（R32）
│   ├── 机器人自动上报错误码
│   ├── 教师手动补充故障描述
│   └── 后台展示故障描述和建议处理方式
└── 维修流程
    ├── 现场可修复 → 当天修复，重新入库
    ├── 需送修/更换配件 → 2 个工作日内完成
    └── 无法修复 → 申请报废，补充新设备
```
