# 迭代路线图

> 一堂五步法 · 第二步：解决方案
> 修炼水平：L5（有快速迭代）→ L6（有迁移能力）

---

## 1. 迭代方法论

### 1.1 Build-Measure-Learn 循环

每个 Phase 遵循精益创业的 Build-Measure-Learn 循环，通过最小可行产品验证核心假设，用数据驱动决策。

```
Build-Measure-Learn 循环
│
├── Build（构建）— 按 P0 → P1 → P2 优先级交付，最小成本验证假设
├── Measure（度量）— 十大典型指标追踪 + Go/No-Go 决策点
└── Learn（学习）— 未验证的假设不进入下一阶段，失败快速暴露
```

### 1.2 核心原则

**"不做没有验证的扩张"**——每个 Phase 的 Go/No-Go 决策点必须通过后才启动下一阶段。

| 原则 | 说明 | 反例（必须避免） |
|------|------|-----------------|
| 假设先行 | 每个 Phase 开始前明确要验证的假设 | "先做出来再说" |
| 数据决策 | Go/No-Go 由数据决定，不由直觉决定 | "感觉市场应该可以" |
| 快速暴露 | 尽早在真实场景中暴露问题 | 躲在实验室里打磨半年 |
| 单点验证 | 先在 2-3 家机构验证，再扩展到 10+ | 一次性铺开 10 家机构 |

### 1.3 财务约束

迭代节奏必须匹配财务能力。3 年累计收入 416 万，Year 1-2 亏损期共需启动资金 25-35 万，Year 3 起自给自足。

| 年份 | 机构数 | 收入 | 净利润 | 阶段 |
|------|--------|------|--------|------|
| Year 1 | 3 所 | 24 万 | -12.3 万 | 种子期（投入） |
| Year 2 | 10 所 | 102 万 | -3 万 | 验证期（接近盈亏平衡） |
| Year 3 | 25 所 | 290 万 | +56 万 | 扩张期（盈利） |

## 2. Phase 1：MVP（0-6 个月）

**目标**：完成核心功能开发，在 2-3 家试点机构完成验证，确认产品价值和商业模式可行性。

### 2.1 验证假设

| # | 核心假设 | 验证指标 | 决策点 |
|---|----------|----------|--------|
| 1 | 初中生对 AI 机器人有持续兴趣 | 完课率 ≥ 80% | 第 4 周 |
| 2 | 教师可以零培训上手使用 | 教师满意度 ≥ 80% | 第 4 周 |
| 3 | 机构引入后出勤率/续课率有可见提升 | 运营数据对比 | 第 16 周 |
| 4 | 学生 2 节课后能独立编程、8 节课后能完成作品 | 作品提交数据 | 第 8 周 |

### 2.2 关键交付物

| 交付物 | 说明 | 优先级 |
|--------|------|--------|
| OTTO 机器人硬件套件 v1 | ESP32-S3 + 8MB PSRAM + ES8311，基础版 ≤ 250 元/台 | P0 |
| 固件核心功能 | 语音唤醒（ESP-SR）、MQTT 通信、舵机控制、OTA 升级 | P0 |
| AI 语音交互 | 接入大模型（Qwen/DeepSeek/GLM），自然语言对话 + 语音控制动作 | P0 |
| 网页端配置工具 | 可视化参数调整、舵机校准、设备管理 | P0 |
| 图形化编程工具 | Blockly 拖拽编程，28+ 预设动作积木 + 3D 模拟预览 | P0 |
| 内置课程体系 | 12 节互动教程（入门 4 节 + 进阶 4 节 + 高级 4 节） | P0 |
| 多角色后台系统 | 管理员 / 教师 / 家长三级权限，内容推送与安全审计 | P0 |
| 机构内部赛平台 | 外观选美 + 动作技能展示（手动组/AI 组）+ 功能创新赛 | P1 |
| DIY 外壳体系 | 开源 STL 模板 + 3 种预设外壳方案 | P1 |
| 教师教案文档 | 配套 12 节课的教师教案和使用指南 | P1 |

### 2.3 十大指标 Phase 1 目标

| 类别 | 指标 | MVP 目标值 | 验证方式 |
|------|------|-----------|----------|
| **获客** | 销转率 | ≥ 30% | 体验课报名 → 正式缴费转化 |
| **获客** | 动销率 | ≥ 70% | 月活跃学生中使用核心功能比例 |
| **获客** | 捕获率 | ≥ 10% | 接触机构中成功签约比例 |
| **服务** | 留存率 | ≥ 50% | 12 个月学生留存 |
| **服务** | 完课率 | ≥ 80% | 报名学生完成全部课时 |
| **服务** | 退款率 | ≤ 5% | 课程退款订单占比 |
| **服务** | 满意率 | ≥ 80% | 教师/学生/家长满意度 |
| **复购** | 复购率 | ≥ 20% | 购买扩展模块的学生比例 |
| **复购** | 续费率 | ≥ 60% | 下一学期续费比例 |
| **复购** | 推荐率 | ≥ 10% | 老带新/机构推荐新机构 |

### 2.4 成功标准

2-3 家试点机构、100+ 学生、教师满意度 ≥ 80%、系统中断 < 1 次/月/机构、出勤率有可见提升。

### 2.5 Go/No-Go 决策点

| 决策点 | 时间 | Go 标准 | No-Go 应对 |
|--------|------|---------|------------|
| 学习曲线验证 | 第 4 周 | ≥ 70% 学生可独立完成基础操作 | 简化交互、增加引导教程 |
| 作品产出验证 | 第 8 周 | ≥ 60% 学生提交至少 1 件完整作品 | 降低难度、增加模板 |
| 机构价值验证 | 第 16 周 | 续约意向 ≥ 2 家、出勤率提升可见 | 深度访谈、调整方向、不进入 Phase 2 |

## 3. Phase 2：规模化（7-18 个月）

**目标**：扩大机构覆盖至 10+ 家，上线机构间联赛，建立区域品牌影响力，实现从"单点验证"到"模式复制"。

### 3.1 验证假设

| # | 核心假设 | 验证指标 | 决策点 |
|---|----------|----------|--------|
| 1 | 竞赛模式能跨机构运行 | 联赛参与机构 ≥ 10 家 | 第 12 月 |
| 2 | BD 模型可复制 | 捕获率 ≥ 15% | 第 12 月 |
| 3 | 师资供给跟得上 | 单校培训周期 ≤ 2 周 | 第 18 月 |

### 3.2 关键交付物

| 交付物 | 说明 | 优先级 |
|--------|------|--------|
| 机构间联赛平台 | 不同机构线上/线下竞赛，排行榜和积分体系 | P0 |
| 扩展模块支持 | 摄像头、大屏显示、额外舵机等即插即用模块 | P0 |
| AI 辅助动作生成 | 自然语言描述 → AI 生成舵机序列，可手动微调 | P1 |
| 音乐配合功能 | 动作与音乐节拍对齐，本地播放 | P1 |
| 数据分析看板 | 机构运营数据、学生参与度趋势、出勤率分析 | P0 |
| 多人协作优化 | 用户快速切换、课堂模式增强、使用时长智能提醒 | P1 |
| 扩展课程体系 | 竞赛集训课程、进阶 DIY 课程 | P1 |
| 硬件 v2 迭代 | 根据试点反馈优化硬件设计，降低故障率 | P2 |

### 3.3 十大指标 Phase 2 目标

| 类别 | 指标 | Phase 2 目标值 | 较 Phase 1 提升 |
|------|------|---------------|-----------------|
| **获客** | 销转率 | ≥ 45% | +15pp |
| **获客** | 动销率 | ≥ 75% | +5pp |
| **获客** | 捕获率 | ≥ 15% | +5pp |
| **服务** | 留存率 | ≥ 60% | +10pp |
| **服务** | 完课率 | ≥ 85% | +5pp |
| **服务** | 退款率 | ≤ 4% | -1pp |
| **服务** | 满意率 | ≥ 85% | +5pp |
| **复购** | 复购率 | ≥ 30% | +10pp |
| **复购** | 续费率 | ≥ 70% | +10pp |
| **复购** | 推荐率 | ≥ 15% | +5pp |

### 3.4 成功标准

10+ 机构、1000+ 学生、首次联赛举办、市级竞赛获奖、系统并发 150-200 台。

### 3.5 Go/No-Go 决策点

| 决策点 | 时间 | Go 标准 | No-Go 应对 |
|--------|------|---------|------------|
| 联赛可行性 | 第 12 月 | ≥ 5 家机构完成首轮联赛 | 简化赛制、推迟规模化 |
| 规模化可行性 | 第 18 月 | 10+ 机构、捕获率 ≥ 15%、续费率 ≥ 70% | 优化 BD 流程、深耕现有机构 |

## 4. Phase 3：生态建设（19-36 个月）

**目标**：构建线上创客社区，实现课程授权输出，从"产品"升级为"平台 + 生态"。

### 4.1 验证假设

| # | 核心假设 | 验证指标 | 验证方式 |
|---|----------|----------|----------|
| 1 | 课程授权模式可行 | 授权收入占总收入 ≥ 20% | 财务数据 |
| 2 | 师训体系可标准化 | 认证教师通过率 ≥ 80% | 师训考核记录 |
| 3 | 多城市可复制 | 新城市 6 个月内实现盈亏平衡 | 城市运营数据 |

### 4.2 关键交付物

| 交付物 | 说明 | 优先级 |
|--------|------|--------|
| 线上创客社区 | 学生上传/下载/点赞/评论作品，社区运营 | P0 |
| 课程授权体系 | 向外部机构输出课程授权，合作 100+ 学校/机构 | P0 |
| 师资培训认证 | 建立师资培训和认证体系，标准化教师输出 | P0 |
| 开发者生态 | 第三方扩展模块支持，社区贡献机制 | P2 |
| 多城市扩展 | 从杭州向周边城市扩展，输出课程和运营模式 | P1 |
| 品牌升级 | 从"性价比"到"专业"的品牌形象升级 | P1 |
| 竞赛品牌化 | 打造 OTTO 专属竞赛 IP，形成年度赛事 | P1 |

## 5. 迁移能力（L6）

> L5 解决"如何快速迭代"，L6 解决"如何从一个成功案例扩展到多个城市"。

### 5.1 课程授权输出能力

**标准化课程包**：教案（Markdown + PDF）+ 课件（PPT + 视频）+ 评估标准（评分表 + 在线测评）+ 教师指南（手册 + 视频）。

**师训 SOP**（招聘 → 独立上岗，约 1-3 个月）：

```
师资培养全流程
│
├── 1. 招聘筛选 — 理工科背景 + 教学经验优先，模拟授课 15 分钟
├── 2. 集中培训（1-2 周）— 硬件操作、编程工具、课程体系、课堂管理
├── 3. 试讲考核 — 观摩 ≥ 4 节课 + 独立试讲 ≥ 2 节，评分 ≥ 80 分通过
├── 4. 独立上岗 — 首月每周巡检，每月教学复盘
└── 5. 持续考核 — 季度教学质量评估，续费率挂钩
```

**质量管控体系**

| 管控维度 | 频率 | 方法 | 负责人 |
|----------|------|------|--------|
| 教学巡检 | 每月 1 次 | 督导到堂听课，填写巡检表 | 教学主管 |
| 学生反馈 | 每学期 2 次 | 问卷调查 + 随机访谈 | 运营团队 |
| 竞赛成绩 | 每学期 1 次 | 参赛率、获奖率统计 | 竞赛负责人 |
| 教师考核 | 每季度 1 次 | 教学评分 + 学生续费率 | 教学主管 |

### 5.2 多城市复制 SOP

**城市开拓清单**（约 6-12 个月从接触到独立运营）：

| 步骤 | 时间 | 关键动作 |
|------|------|----------|
| 1. 市场调研 | 2-4 周 | 当地教培机构分布、竞品定价、目标学校名单（10-20 家） |
| 2. BD 拜访 | 4-8 周 | 教育展会、预约拜访、产品演示 + 案例数据，目标签约 2-3 家 |
| 3. 试点签约 | 2-4 周 | 签订协议、硬件部署、教师培训（1-2 周）、首批课程启动 |
| 4. 运营支持 | 3-6 个月 | 驻场支持前 4 节课、每月巡检、培养 Local Champion |
| 5. 独立运营 | 6 个月后 | Local Champion 接手、总部远程支持、本地 BD 自主开拓 |

**单城市盈亏平衡模型**：年收入约 140 万（5 所机构）、成本约 100-120 万、盈亏平衡约 12-18 个月。

**管理半径**：同时运营 ≤ 3 个城市、每城市至少 1 名 Local Champion、总部每月至少 1 次现场支持。

**城市优先级排序**

```
城市扩展路线图
│
├── 第一梯队：杭州（总部所在，品牌认知基础好）
├── 第二梯队：宁波 / 温州（Year 2 下，浙江省内，BD 成本低）
├── 第三梯队：南京 / 苏州（Year 3 上，长三角核心，消费能力强）
├── 第四梯队：成都 / 武汉（Year 3 下，中西部教育重镇，需本地化）
└── 第五梯队：深圳 / 广州（按需，珠三角容量大但竞争激烈）
```

### 5.3 师资培训认证体系

| 等级 | 名称 | 能力要求 | 培训周期 | 认证标准 |
|------|------|----------|----------|----------|
| L1 | 基础教师 | 独立完成 12 节基础课程教学 | 1-2 个月 | 试讲 ≥ 80 分 + 书面考核通过 |
| L2 | 进阶教师 | 教授竞赛集训课程 + 指导学生参赛 | 3-6 个月 | L1 基础上 + 指导学生获奖 ≥ 1 次 |
| L3 | 导师 | 培训新教师 + 参与课程研发 | 6-12 个月 | L2 基础上 + 培训新教师 ≥ 3 名通过 |

### 5.4 硬件供应链标准化

**核心元器件选型**

| 元器件 | 选型方案 | 备选供应商 | 成熟度 |
|--------|----------|-----------|--------|
| 主控芯片 | ESP32-S3-WROOM-1-N8R8 | 乐鑫官方 + 合肥兆易 | 成熟 |
| 音频芯片 | ES8311 | 矽力杰 + 备选 ES8388 | 成熟 |
| 舵机 | SG90 / MG90S | 辉盛 + 威纶 | 成熟 |
| 电池 | 18650 + TP4056 充电管理 | 多供应商可选 | 成熟 |

**代工合作模式**

| 阶段 | 批量 | 模式 | 单台成本目标 |
|------|------|------|-------------|
| Phase 1 | 30-100 台 | 自行组装 + 外协焊接 | ≤ 300 元 |
| Phase 2 | 300-1000 台 | PCBA 外包 + 自行组装 | ≤ 250 元 |
| Phase 3 | 3000+ 台/年 | 整机代工（OEM） | ≤ 200 元 |

## 7. 风险与依赖

| 风险项 | 影响阶段 | 应对措施 |
|--------|----------|----------|
| Otto DIY 开源项目维护停滞 | 全阶段 | 关注社区动态，必要时 Fork 独立维护 |
| 大模型 API 成本上升或不可用 | Phase 1-2 | 支持多模型切换（Qwen/DeepSeek/GLM），不绑定单一供应商 |
| 试点机构验证失败 | Phase 1 | 设置 2-3 家试点分散风险，快速迭代产品 |
| 硬件供应链不稳定 | Phase 1-2 | 核心元器件（ESP32-S3、PSRAM）选择成熟方案，备选供应商 |
| 竞品进入低价位市场 | Phase 2-3 | 加速占领市场，通过开源生态和竞赛体系建立壁垒 |
| 政策变化影响素质教育 | 全阶段 | 产品符合 STEAM 教育政策方向，保持灵活性 |

### 风险优先级矩阵

| 风险 | 发生概率 | 影响程度 | 优先级 | 监控频率 |
|------|----------|----------|--------|----------|
| 试点验证失败 | 中 | 高 | **P0** | 每周 |
| 硬件供应链不稳定 | 中 | 高 | **P0** | 每月 |
| 大模型 API 成本 | 高 | 中 | **P1** | 每月 |
| 竞品低价竞争 | 低 | 高 | **P1** | 每季度 |
| Otto DIY 停滞 | 低 | 中 | **P2** | 每季度 |
| 政策变化 | 低 | 高 | **P2** | 每半年 |

---

> **迭代原则：** 每个 Phase 都有明确的验证假设、交付物、成功标准和 Go/No-Go 决策点。不做没有验证的扩张，不做没有数据的决策。
