## 1. 功能架构总览

PRD 定义了 33 项功能需求（R1-R33），覆盖 8 个功能模块。按层次关系可归纳为 6 个功能层 + 2 个横切关注点：

```
OTTO 123 功能架构
├── 语音交互层（R1-R5）
│   └── AI对话 · 语音控制 · 知识问答 · 离线唤醒
├── 开发工具层（R6-R9, R14-R19）
│   └── 网页配置 · 图形化编程 · 3D预览 · AI辅助创作
├── 硬件扩展层（R10-R13）
│   └── DIY外壳 · 标准化接口 · 即插即用模块
├── 竞赛平台层（R20-R23）
│   └── 内部赛 · 联赛 · 社区 · 评分模板
├── 权限管控层（R24-R27）
│   └── 三级权限 · 内容推送 · 安全审计 · 使用监控
└── 课堂管理层（R28-R30, R31-R33）
    └── 多人切换 · 课堂模式 · 设备运维
```

横切关注点贯穿所有功能层：

| 横切关注点 | 覆盖范围 | 说明 |
|------------|----------|------|
| 非功能性约束 | 全模块 | 性能（语音唤醒<500ms）、可用性（99.5%）、安全（TLS 1.2+ + AES-256） |
| 通信架构 | 全模块 | MQTT（控制通道）+ WebSocket（媒体/音频流） |

### 3.2 依赖风险分析

| 依赖链 | 风险等级 | 风险描述 | 缓解措施 |
|--------|----------|----------|----------|
| R1→R2→R14→R20 | 高 | 语音交互是编程和竞赛的基础，任一环节卡住会阻塞后续 | R5 离线唤醒作为降级方案；R14 可先支持纯编程不含语音 |
| R6→R7→R14 | 中 | 舵机校准直接影响编程体验 | 提供默认校准值，校准工具可并行开发 |

| R20→R21→R22 | 低 | 竞赛链路按阶段推进，不会阻塞 MVP | 三期逐步推进，每期独立验证 |

### 4.2 通信架构

```
┌──────────┐     MQTT（控制通道）     ┌──────────┐
│          │  ← 动作指令/OTA/状态上报 → │          │
│  OTTO    │                           │  云端    │
│  Robot   │     WebSocket（媒体通道）  │  服务    │
│          │  ← 语音流/编程数据同步 →   │          │
└──────────┘                           └──────────┘
     │                                      │
     │  本地执行                              │
     ├─ ESP-SR 离线唤醒                      ├─ 大模型 API
     ├─ 舵机控制                             ├─ TTS/ASR 服务
     ├─ 传感器读取                           ├─ 动作存储
     └─ 音频播放（本地缓存）                   └─ 数据分析
```

**通道分离原则**：

| 通道 | 协议 | 用途 | 频率 | 延迟要求 |
|------|------|------|------|----------|
| 控制通道 | MQTT | 动作指令、状态上报、OTA 触发 | 低频 | < 500ms |
| 媒体通道 | WebSocket | 语音流、编程数据实时同步 | 高频 | < 200ms |

**课堂网络优化**：

- 推荐 5GHz 独立 AP，单课堂支持 15-20 台设备
- 非活跃设备降低心跳频率
- 常用 TTS 音频本地缓存，减少云端请求

### 5.3 DIY 外壳体系

| 外壳风格 | 描述 | 难度 | 适用场景 |
|----------|------|------|----------|
| 基础款 | Otto DIY 标准版，开源 STL | 入门 | L1 入门课程、快速上手 |
| 机甲风 | 金属质感配色，棱角分明 | 中等 | L2-L3 竞赛、外观选美赛 |
| 可爱风 | 圆润造型，明亮配色 | 入门 | L1 低龄学生、女生偏好 |
| 卡通 IP 风 | 动物/角色造型，可定制 | 中等 | 创意项目、外观选美赛 |

**外壳体系设计原则**：

- 所有 STL 文件开源，机构/学生可自由修改
- 外壳安装后固件自动检测舵机是否受阻或传感器是否被遮挡
- 异常时提示检查外壳安装，确保安全
- 鼓励学生在基础模板上二次创作，形成差异化作品

### 5.4 器材配置与课程级别对应

| 课程级别 | 目标年龄 | 核心硬件配置 | 套件成本 | 课时量 |
|----------|----------|-------------|----------|--------|
| L1 入门 | 8-10 岁 | 入门套件（基础配置） | 800-1,200 元 | 16-32 课时 |
| L2 进阶 | 10-12 岁 | 入门套件 + 摄像头 + 额外舵机 | 1,500-2,500 元 | 32-48 课时 |
| L3 扩展 | 12-14 岁 | 进阶套件 + 显示屏 + 多传感器 | 2,500-4,000 元 | 48-64 课时 |
| L4 专业 | 14-16 岁 | 全配置 + LiDAR/深度相机 | 5,000-10,000 元 | 64-96 课时 |

**成本控制要点**：

- L1 入门套件控制在 250 元硬件成本以内（不含扩展模块）
- L2-L3 通过模块化扩展，避免重复购买基础套件
- L4-L5 可引入第三方硬件（如 LiDAR、深度相机），通过标准化接口集成
- 课堂配比 2-3 人/台，进一步摊薄人均硬件成本

### 6.3 内置课程体系（R17）

R17 定义了 12 节平台内置课程，分为三个级别：

| 级别 | 课程数 | 核心内容 | 教学目标 |
|------|--------|----------|----------|

**课程呈现形式**：平台内置互动教程，每课结构为「目标 → 引导 → 实操 → 挑战」，配套教师教案文档。

## 7. 非功能性要求概览

### 7.1 性能要求

| 指标 | 目标 | 影响模块 |
|------|------|----------|
| 语音唤醒响应 | < 500ms | R5（离线唤醒） |
| 语音指令识别 + 执行 | < 2 秒（P95） | R2（语音控制） |
| AI 对话首字响应 | < 2 秒（P95），流式 LLM + 流式 TTS | R1（AI 对话） |
| 动作编程预览 | < 1 秒（点击到机器人开始执行） | R15（编程预览） |
| OTA 升级 | 单台 < 5 分钟，支持分批并发 | R8（OTA 升级） |
| 云端并发 | 10 个机构（150-200 台设备）同时在线，P95 延迟不超标 | 全模块 |

### 7.2 可用性要求

| 指标 | 目标 | 降级策略 |
|------|------|----------|
| 云端服务可用性 | >= 99.5%（月度） | 机器人进入离线模式，保留基础动作和预设对话 |
| 计划外停机 | 自动降级 | 离线唤醒 + 预设动作指令可用 |
| 故障恢复 RTO | < 4 小时 | 自动故障转移 + 快速回滚 |
| 数据 RPO | < 1 小时（学生作品和对话摘要） | 定时备份 + 异地冗余 |

### 7.3 安全与隐私

| 要求 | 具体措施 | 影响模块 |
|------|----------|----------|
| 数据存储 | 学生数据存储于国内云服务器 | 全模块 |
| 传输加密 | TLS 1.2+ | 全模块通信 |
| 存储加密 | AES-256 | 用户数据、配置信息 |
| 信息收集最小化 | 仅收集昵称、年级、所属班级；不收集真实姓名、身份证号、家庭住址 | R24-R27 |
| 账号安全 | 密码 >= 8 位，连续 5 次失败锁定 30 分钟，教师/管理员支持双因素认证 | R24 |
| 安全告警范围 | 仅保留触发关键词前后各 2 轮对话，完整对话不存储不展示 | R26 |

### 7.4 通信架构

| 维度 | 控制通道 | 媒体通道 |
|------|----------|----------|
| 协议 | MQTT | WebSocket |
| 用途 | 动作指令、状态上报、OTA 触发 | 语音流、编程数据实时同步 |
| 频率 | 低频 | 高频 |
| 延迟要求 | < 500ms | < 200ms |
