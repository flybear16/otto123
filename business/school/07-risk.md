## 七、风险管控

### 7.1 风险矩阵

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| **学校中途终止合作** | 中 | 高 | 合同明确违约责任；每学期评估续签；不押重资产在校 |
| **招生不足** | 中 | 高 | 体验课引流；学校强制摊派（慎用）；保底招生承诺 |
| **师资流失** | 中 | 中 | 标准化教案降低依赖；备选师资池；有竞争力的课酬 |
| **设备批量故障** | 低 | 高 | 20%备用机；供应商质保协议；现场快速更换流程 |
| **家长投诉** | 中 | 中 | 明确服务边界；及时响应机制；不满意可退剩余课时 |
| **竞品低价抢单** | 高 | 中 | 强化差异化（开源、AI、竞赛）；绑定长期合作 |
| **政策变化** | 低 | 高 | 关注政策动向；合规经营；多元化渠道（不只依赖学校） |

### 7.2 应急预案

**场景1: 授课老师临时请假**
```
Level 1（提前1天知晓）:
→ 启动备选老师，提前熟悉教案
→ 通知学校调整人员

Level 2（当天突发）:
→ 协调学校信息技术老师代课（提供简化版教案）
→ 或改为"视频课+线上答疑"
→ 下次课补课并赠送小礼品补偿
```

**场景2: 设备批量故障**
```
Level 1（单台故障）:
→ 现场更换备用机
→ 故障机带回维修

Level 2（多台故障，课程无法继续）:
→ 立即改为"理论课+视频演示"
→ 安排补课时间
→ 向家长说明情况并致歉
```

**场景3: 家长集体退费诉求**
```
Level 1（个别不满）:
→ 单独沟通，了解具体问题
→ 提供补课/换班/部分退费选项

Level 2（集体投诉）:
→ 启动危机公关小组
→ 公开说明情况并提出解决方案
→ 必要时全额退费止损保口碑
```
