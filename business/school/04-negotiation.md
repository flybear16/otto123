## 四、商务谈判策略

### 4.1 决策链分析

```
最终决策人: 校长
    ├── 分管副校长（教学/德育）
    │       └── 教务处主任
    │               └── 信息技术/科学教研组长
    └── 家委会代表（影响付费意愿）

关键人识别:
• 支持者: 信息技术老师（想减轻负担）、科技教育负责人（要政绩）
• 中立者: 教务主任（关心流程合规）、财务（关心收费模式）
• 反对者: 已有合作机构（利益冲突）、保守派教师（怕麻烦）
```

### 4.2 分角色沟通策略

**对校长: 谈战略价值**
```
"校长，咱们学校科技教育这块，现在是什么定位？
（倾听）...
我们这套方案，能帮学校快速打造科技特色，
一学期就能出成果、拿奖项、上报道。
而且零投入、零风险，家长满意度还高。
您看这周能不能安排个20分钟，我给您详细演示一下？"
```

**对教务主任: 谈执行便利**
```
"主任，我们理解学校最担心的是安全和流程。
我们全套方案都是合规的：
• 师资都有教师资格证或相关资质证明
• 每节课都有教案、考勤表、安全预案
• 保险全覆盖，责任界定清晰
• 家长缴费直接到我们账户，学校不涉及收费
您只需要提供场地和协助通知，其他我们全包。"
```

**对信息技术老师: 谈减负增值**
```
"老师，听说您现在带社团课？挺辛苦的吧。
我们来了之后，您不用自己备课、不用自己教，
只需要协助我们管理一下课堂纪律就行。
而且我们课程都是标准化的，您也能学到最新的机器人技术，
以后评职称、写论文都有素材。
另外，我们每学期会给协助老师一定的课时补贴。"
```

**对家长: 谈性价比和成果**
```
"家长您好，咱们这个社团课，一学期16节课，1280元，
平均一节课80元，比外面便宜一半还多。
而且孩子能带走一套价值300多元的机器人，
学期末还有作品展示、竞赛评比、结业证书。
最关键的是，在学校上，您不用接送，省心！"
```

### 4.3 谈判话术库

**场景1: 学校说"我们已经有编程课了"**
```
回应: "太好了，说明学校重视科技教育。
我们OTTO机器人课和纯编程课是互补的：
• 编程课是'软件思维'，我们是'硬件+软件'，孩子能摸到实物
• 编程课是'屏幕学习'，我们是'动手创造'，培养工程能力
• 很多学校都是两者结合，编程课打基础，机器人课做项目
您看能不能先开一期体验课，让孩子和家长自己选？"
```

**场景2: 学校说"家长可能不愿意付费"**
```
回应: "理解您的顾虑。我们做过调研，
现在家长对科技类课程付费意愿很强，
关键是看性价比和成果展示。
我们建议先做一场免费体验课，
让孩子现场组装机器人、现场展示给家长看，
用效果说话。如果报名人数不足20人，我们自担风险，不收费。"
```

**场景3: 学校说"需要上会讨论"**
```
回应: "应该的，这是大事。我们准备一份详细的合作方案，
包括课程大纲、师资介绍、安全保障、合作案例，
您看什么时候方便，我过来给领导们做个15分钟汇报？
另外，我们可以提供1-2所已合作学校的联系方式，
方便您侧面了解。"
```

**场景4: 学校说"分成比例能不能调整"**
```
回应: "校长，70%是我们经过成本核算的底线：
• 硬件设备折旧和损耗
• 专业师资课时费
• 课程研发和迭代
• 运营管理和售后
我们愿意在招生人数上做承诺：
如果一学期招满30人，学校分成比例可以提升到35%。
您看这样行不行？"
```
