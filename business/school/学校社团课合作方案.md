## 一、市场机会分析

### 1.1 政策红利窗口期

| 政策文件 | 核心要点 | 机会点 |
|----------|----------|--------|
| 《义务教育课程方案(2022年版)》 | 科学课程占比提升至8-10%，强调"做中学" | 机器人社团课直接响应政策要求 |
| "双减"政策 | 学科类培训受限，素质类课程需求爆发 | 学校急需优质非学科内容填充 |
| 课后服务全覆盖 | 要求"一校一案"，鼓励引入社会资源 | 第三方机构进校门槛降低 |
| 科技特长生政策 | 多地中考科技特长生加分/降分录取 | 家长对科技类课程付费意愿强 |

### 1.2 学校痛点精准画像

```
【校长层】
痛点1: 课后服务"有但不好"，家长满意度低
痛点2: 特色课程缺师资、缺内容、缺成果展示
痛点3: 科技教育投入大（设备+场地+师资），预算有限

【教务层】
痛点1: 社团课老师流动性大，课程 continuity 差
痛点2: 外聘机构良莠不齐，筛选成本高
痛点3: 安全管理责任大，担心出事

【教师层】
痛点1: 带社团课无额外报酬或报酬低，积极性差
痛点2: 非专业出身，面对机器人/编程力不从心
痛点3: 课时被占用，影响主科教学

【家长层】
痛点1: 学校社团课"水"，学不到真东西
痛点2: 校外培训贵（乐高/编程200-300元/课时）
痛点3: 接送麻烦，希望学校一站式解决
```

### 1.3 竞品格局分析

| 竞品类型 | 代表品牌 | 价格区间 | 劣势 | 我们的机会 |
|----------|----------|----------|------|------------|
| 国际大牌 | 乐高教育 | ¥300-500/课时 | 太贵、设备封闭、需专用教室 | 性价比+开源可扩展 |
| 国产连锁 | 编程猫、核桃编程 | ¥150-250/课时 | 纯软件/线上为主，缺乏硬件动手 | 软硬结合+实体机器人 |
| 本地小机构 | 各类创客工作室 | ¥100-200/课时 | 不系统、师资不稳定、无品牌 | 标准化课程+品牌背书 |
| 学校自办 | 信息技术老师带 | ¥0-50/课时 | 老师不专业、设备老旧、内容单一 | 专业输出+完整方案 |

**结论**: 中端市场（80-150元/课时）存在空白，OTTO方案定位精准。

## 二、合作模式设计

### 2.1 三种合作模式对比

| 模式 | 学校投入 | 我方投入 | 收益分配 | 适合场景 | 推荐指数 |
|------|----------|----------|----------|----------|----------|
| **A. 纯服务外包** | 场地+组织 | 设备+师资+课程 | 学校付服务费¥8-12万/学期 | 学校预算充足，求省心 | ⭐⭐⭐⭐ |
| **B. 联合运营** | 场地+部分设备 | 课程+师资+运营 | 学费分成（学校30%，我方70%） | 学校愿意共担风险共享收益 | ⭐⭐⭐⭐⭐ |
| **C. 课程授权** | 场地+设备+本校老师 | 课程授权+师训 | 一次性授权费¥3-5万+年费¥1-2万 | 学校有师资，预算有限 | ⭐⭐⭐ |

### 2.2 主推模式：联合运营（B模式）

**核心逻辑**: 学校出场地和部分设备（已有电脑教室即可），我方输出完整课程和师资，学费按比例分成。

**为什么主推此模式**:
1. **学校接受度高**: 零现金投入，风险低
2. **我方现金流好**: 直接向家长收费，账期短
3. **利益绑定深**: 学校有动力配合招生和运营
4. **可复制性强**: 标准化后快速推广多校

**合作框架**:
```
┌─────────────────────────────────────────────────────┐
│                   联合运营合作协议                    │
├─────────────────────────────────────────────────────┤
│  甲方（学校）               乙方（我方）              │
│  ─────────────             ─────────────            │
│  • 提供社团课场地（计算机教室/创客空间）              │
│  • 提供现有电脑设备（学生自带或学校提供）             │
│  • 协助招生（发通知、家长会宣传）                    │
│  • 协助管理（考勤、安全、家长沟通）                  │
│                            • 提供OTTO机器人设备      │
│                            • 派遣专业师资授课        │
│                            • 提供完整课程体系        │
│                            • 负责课程运营和售后      │
├─────────────────────────────────────────────────────┤
│  收益分配: 学费总收入 × 学校30% + 我方70%            │
│  结算周期: 每月结算一次                              │
│  合作期限: 首签1学期，续签优先                       │
└─────────────────────────────────────────────────────┘
```

### 2.3 合作学校筛选标准

| 维度 | 硬性要求 | 加分项 |
|------|----------|--------|
| **学校类型** | 小学（3-6年级）或初中（7-8年级） | 科技特色学校、STEM示范校 |
| **学生规模** | 单年级≥200人 | 总人数≥1000人 |
| **场地条件** | 有计算机教室或可用教室≥1间 | 已有创客空间/机器人教室 |
| **家长付费能力** | 学区房价≥当地均价1.2倍 | 周边有高端社区 |
| **决策效率** | 能在2周内完成内部决策 | 校长直接分管科技教育 |
| **合作意愿** | 明确表达引入第三方意愿 | 已有其他素质类合作先例 |

**黑名单（不合作）**:
- 要求我方垫资采购学校设备
- 要求学费100%先进学校账户再分配
- 历史上有拖欠合作方款项记录
- 地理位置偏远（单程通勤>1小时）

## 四、商务谈判策略

### 4.1 决策链分析

```
最终决策人: 校长
    ├── 分管副校长（教学/德育）
    │       └── 教务处主任
    │               └── 信息技术/科学教研组长
    └── 家委会代表（影响付费意愿）

关键人识别:
• 支持者: 信息技术老师（想减轻负担）、科技教育负责人（要政绩）
• 中立者: 教务主任（关心流程合规）、财务（关心收费模式）
• 反对者: 已有合作机构（利益冲突）、保守派教师（怕麻烦）
```

### 4.2 分角色沟通策略

**对校长: 谈战略价值**
```
"校长，咱们学校科技教育这块，现在是什么定位？
（倾听）...
我们这套方案，能帮学校快速打造科技特色，
一学期就能出成果、拿奖项、上报道。
而且零投入、零风险，家长满意度还高。
您看这周能不能安排个20分钟，我给您详细演示一下？"
```

**对教务主任: 谈执行便利**
```
"主任，我们理解学校最担心的是安全和流程。
我们全套方案都是合规的：
• 师资都有教师资格证或相关资质证明
• 每节课都有教案、考勤表、安全预案
• 保险全覆盖，责任界定清晰
• 家长缴费直接到我们账户，学校不涉及收费
您只需要提供场地和协助通知，其他我们全包。"
```

**对信息技术老师: 谈减负增值**
```
"老师，听说您现在带社团课？挺辛苦的吧。
我们来了之后，您不用自己备课、不用自己教，
只需要协助我们管理一下课堂纪律就行。
而且我们课程都是标准化的，您也能学到最新的机器人技术，
以后评职称、写论文都有素材。
另外，我们每学期会给协助老师一定的课时补贴。"
```

**对家长: 谈性价比和成果**
```
"家长您好，咱们这个社团课，一学期16节课，1280元，
平均一节课80元，比外面便宜一半还多。
而且孩子能带走一套价值300多元的机器人，
学期末还有作品展示、竞赛评比、结业证书。
最关键的是，在学校上，您不用接送，省心！"
```

### 4.3 谈判话术库

**场景1: 学校说"我们已经有编程课了"**
```
回应: "太好了，说明学校重视科技教育。
我们OTTO机器人课和纯编程课是互补的：
• 编程课是'软件思维'，我们是'硬件+软件'，孩子能摸到实物
• 编程课是'屏幕学习'，我们是'动手创造'，培养工程能力
• 很多学校都是两者结合，编程课打基础，机器人课做项目
您看能不能先开一期体验课，让孩子和家长自己选？"
```

**场景2: 学校说"家长可能不愿意付费"**
```
回应: "理解您的顾虑。我们做过调研，
现在家长对科技类课程付费意愿很强，
关键是看性价比和成果展示。
我们建议先做一场免费体验课，
让孩子现场组装机器人、现场展示给家长看，
用效果说话。如果报名人数不足20人，我们自担风险，不收费。"
```

**场景3: 学校说"需要上会讨论"**
```
回应: "应该的，这是大事。我们准备一份详细的合作方案，
包括课程大纲、师资介绍、安全保障、合作案例，
您看什么时候方便，我过来给领导们做个15分钟汇报？
另外，我们可以提供1-2所已合作学校的联系方式，
方便您侧面了解。"
```

**场景4: 学校说"分成比例能不能调整"**
```
回应: "校长，70%是我们经过成本核算的底线：
• 硬件设备折旧和损耗
• 专业师资课时费
• 课程研发和迭代
• 运营管理和售后
我们愿意在招生人数上做承诺：
如果一学期招满30人，学校分成比例可以提升到35%。
您看这样行不行？"
```

## 五、执行落地SOP

### 5.1 从接触到开课全流程

```
Week 1-2: 意向建立
├── Day 1-3: 初步接触（电话/拜访）
│   └── 输出: 学校信息表、决策链地图
├── Day 4-7: 需求调研（访谈关键人）
│   └── 输出: 学校痛点清单、合作意愿评估
└── Day 8-14: 方案定制（根据学校特点调整）
    └── 输出: 定制化合作方案PPT

Week 3-4: 谈判签约
├── Day 15-18: 方案汇报（给校长/决策层）
│   └── 输出: 会议纪要、待确认事项清单
├── Day 19-22: 合同谈判（法务审核）
│   └── 输出: 合作协议终稿
└── Day 23-28: 正式签约
    └── 输出: 签约合同、启动会安排

Week 5-6: 开课准备
├── Day 29-32: 师资安排（确定授课老师）
├── Day 33-36: 设备准备（采购/调试OTTO套件）
├── Day 37-40: 招生启动（学校协助发通知）
└── Day 41-42: 家长说明会（现场答疑、缴费）

Week 7: 正式开课
├── Day 43+: 首课交付
└── 持续: 每周课程、每月反馈、学期总结
```

### 5.2 关键节点检查清单

**签约前检查清单**:
- [ ] 学校资质核实（办学许可证、法人身份证）
- [ ] 场地实地考察（面积、设备、安全出口）
- [ ] 决策人背调（过往合作口碑、决策风格）
- [ ] 合同条款审核（特别是退出机制和违约责任）
- [ ] 竞品情况了解（学校是否已有类似合作）

**开课前检查清单**:
- [ ] 师资培训完成（熟悉教案、设备调试）
- [ ] 设备到位并测试（每套OTTO预组装检查）
- [ ] 招生物料制作（海报、手册、报名链接）
- [ ] 家长群建立（提前沟通、答疑）
- [ ] 应急预案准备（设备故障、学生受伤、突发停课）

**首课检查清单**:
- [ ] 提前30分钟到场布置
- [ ] 设备二次检查（每套通电测试）
- [ ] 学生签到表、家长联系方式确认
- [ ] 拍照/录像准备（用于后续宣传）
- [ ] 课后反馈问卷（了解满意度）

### 5.3 日常运营SOP

**每周运营节奏**:
```
周一:
├── 上午: 本周课程准备（教案复习、设备检查）
```

## 六、财务模型与定价

### 6.1 单校财务模型（以30人班为例）

**收入端（每学期）**:
| 项目 | 单价 | 人数 | 金额 |
|------|------|------|------|
| 社团基础班学费 | ¥1,280 | 30 | ¥38,400 |
| 社团进阶班学费 | ¥1,580 | 20（转化率66%） | ¥31,600 |
| **学期总收入** | - | - | **¥70,000** |

**成本端（每学期）**:
| 项目 | 金额 | 说明 |
|------|------|------|
| 硬件成本 | ¥9,000 | 30套×¥300（含备用） |
| 师资成本 | ¥12,000 | 16周×2次×¥150课时费 |
| 学校分成 | ¥21,000 | 总收入¥70,000×30% |
| 运营杂费 | ¥3,000 | 交通、物料、保险等 |
| **学期总成本** | **¥45,000** | - |

**利润**:
```
学期毛利: ¥70,000 - ¥45,000 = ¥25,000
毛利率: 35.7%
单校年利润（2学期）: ¥50,000
```

### 6.2 规模化财务预测

| 规模 | 学校数 | 单校年均利润 | 年总利润 | 所需师资 |
|------|--------|--------------|----------|----------|
| 试点期 | 3校 | ¥50,000 | ¥150,000 | 2人 |
| 扩张期 | 10校 | ¥50,000 | ¥500,000 | 5人 |
| 成熟期 | 30校 | ¥45,000* | ¥1,350,000 | 12人 |

*成熟期单校利润略降原因：管理成本增加、竞争加剧需降价

### 6.3 定价策略详解

**为什么定价¥1,280/学期（16课时）**:

| 对比维度 | 乐高WeDo | 编程猫 | 大疆 | OTTO方案 |
|----------|----------|--------|------|----------|
| 价格 | ¥2,500+/期 | ¥1,800+/期 | ¥3,500+/期 | ¥1,280/期 |
| 课时 | 12-16 | 10-12 | 16 | 16 |
| 单价/课时 | ¥156-208 | ¥150-180 | ¥219 | ¥80 |
| 硬件归属 | 机构所有 | 无硬件 | 机构所有 | 学生带走 |
| 性价比 | 低 | 中 | 低 | **高** |

**家长心理价位测试**:
- ¥1,000以下: 觉得"便宜可能没好货"
- ¥1,000-1,500: "合理价位，可以试试"
- ¥1,500-2,000: "有点贵，但值得考虑"
- ¥2,000以上: "太贵了，不如去外面报"

**结论**: ¥1,280处于"合理偏低"区间，既保证利润又有竞争力。

## 七、风险管控

### 7.1 风险矩阵

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| **学校中途终止合作** | 中 | 高 | 合同明确违约责任；每学期评估续签；不押重资产在校 |
| **招生不足** | 中 | 高 | 体验课引流；学校强制摊派（慎用）；保底招生承诺 |
| **师资流失** | 中 | 中 | 标准化教案降低依赖；备选师资池；有竞争力的课酬 |
| **设备批量故障** | 低 | 高 | 20%备用机；供应商质保协议；现场快速更换流程 |
| **家长投诉** | 中 | 中 | 明确服务边界；及时响应机制；不满意可退剩余课时 |
| **竞品低价抢单** | 高 | 中 | 强化差异化（开源、AI、竞赛）；绑定长期合作 |
| **政策变化** | 低 | 高 | 关注政策动向；合规经营；多元化渠道（不只依赖学校） |

### 7.2 应急预案

**场景1: 授课老师临时请假**
```
Level 1（提前1天知晓）:
→ 启动备选老师，提前熟悉教案
→ 通知学校调整人员

Level 2（当天突发）:
→ 协调学校信息技术老师代课（提供简化版教案）
→ 或改为"视频课+线上答疑"
→ 下次课补课并赠送小礼品补偿
```

**场景2: 设备批量故障**
```
Level 1（单台故障）:
→ 现场更换备用机
→ 故障机带回维修

Level 2（多台故障，课程无法继续）:
→ 立即改为"理论课+视频演示"
→ 安排补课时间
→ 向家长说明情况并致歉
```

**场景3: 家长集体退费诉求**
```
Level 1（个别不满）:
→ 单独沟通，了解具体问题
→ 提供补课/换班/部分退费选项

Level 2（集体投诉）:
→ 启动危机公关小组
→ 公开说明情况并提出解决方案
→ 必要时全额退费止损保口碑
```
